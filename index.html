Learning Git

Learn how Git can help you collaborate, manage code efficiently, and become a better developer.

What is Git?

Git is a distributed version control system used by developers to track changes in source code during software development. It's a key tool in the DevOps workflow. It was created by Linus Torvalds in 2005. Some of the Git hosting platforms are: GitHub, GitLab and Bitbucket

Why Learn Git?

  • Track changes in your codebase.
  • Collaborate efficiently with teams.
  • Revert to previous versions when needed.

Workflow

This is the standard Git workflow to manage your version-controlled project:

  1. Initialization - git init
  2. Configuration -
    git config user.name "YourGitHubUsername"
    git config user.email "YourGitHubEmail"
  3. Adding - git add <filename>
  4. Committing - git commit -m "Your commit message"
  5. Connecting Repo - git remote add origin <repository-url>
  6. Pushing - git push origin <branchname>

Basic Git Commands

Here are some basic Git commands to get started:

  • git init - Initialize a new repository
  • git status - Check the current status
  • git add . - Stage changes
  • git commit -m "message" - Commit changes
  • git push - Push changes to a remote repository
